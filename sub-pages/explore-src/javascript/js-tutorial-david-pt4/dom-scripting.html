Introduction to JavaScript: 2

What is DOM Scripting?

Most of what we have been doing with JavaScript so far could most accurately be described as DOM Scripting. The DOM, or Document Object Model, is a conceptual model of the document (web page). Essentially it is a heirarchical model of the document markup and content where each HTML element forms a node.

Like many other aspects of the Web, the DOM is maintained by the W3C, and all browsers use the same standard DOM. This means we can use JavaScript to manipulate the DOM (e.g. add and modify elements and content), knowing that what we do will work exactly the same in all browsers.

For example, if we decide that we need to add a new class (or any other attribute) to an unordered list, or if we need to create a new unordered list, the same script that makes that happen will work just as well in Firefox as it will in Chrome or Edge. It wasn't always so simple, but that's another story.

Although JavaScript can do many things, in front-end development it is most commonly used for DOM Scripting. Over the years it has developed some very powerful tools that help us manipulate the DOM and that make it easy. For example, HTML elements can be selected in JavaScript using the same selection syntax as they can in CSS, using the document.querySelector() method. It was introduced in 2013 and browser support is excellent.
